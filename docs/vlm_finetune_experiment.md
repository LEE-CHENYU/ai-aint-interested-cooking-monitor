# VLM Fine-Tuning Experiment: Kitchen Safety Monitor

## Objective

Fine-tune Gemma 3 4B (vision-language model) to classify kitchen cooking states and detect safety hazards from top-down stove images. The model should output structured JSON with dish type, cooking state, safety flag, and reason.

## Setup

| Component | Detail |
|-----------|--------|
| **Base model** | `unsloth/gemma-3-4b-it` (Gemma 3 4B instruction-tuned) |
| **Fine-tuning framework** | Unsloth FastVisionModel + LoRA |
| **Hardware** | NVIDIA RTX PRO 6000 Blackwell (96GB VRAM) on GCP |
| **Training images** | 36 synthetic images generated by Gemini, cropped from 6×6 grid |
| **Image source** | Top-down overhead photos of stove, generated from detailed prompts |

### Dataset: 6 Dishes × 6 States Matrix

**Dishes (rows):** pasta, ramen, mapo_tofu, curry, soup, rice

**States (columns):**

| State | Label | Safe? | Description |
|-------|-------|-------|-------------|
| `1_cold` | cold | Yes | Not cooking yet, no heat |
| `2_simmer` | simmering | Yes | Gentle simmer, under control |
| `3_boil` | boiling | Yes | Active boil but contained in pot |
| `4_BOIL_OVER` | boil_over | **No** | Liquid spilling over pot rim onto stove |
| `5_done_ok` | done | Yes | Cooking complete, looks good |
| `6_FAILED` | burnt | **No** | Food overcooked, burnt or ruined |

36 total images. 24 safe, 12 failure (boil_over + burnt).

### System Prompt

```
You are a kitchen safety monitor. Look at this top-down photo of a stove.
Respond with ONLY a JSON object, no other text:
{"dish": "<what food>", "state": "<cold|simmering|boiling|boil_over|done|burnt>",
 "safe": true/false, "reason": "<10 words max>"}
```

## Training Configuration

| Parameter | Value |
|-----------|-------|
| **LoRA rank (r)** | 16 |
| **LoRA alpha** | 16 |
| **LoRA dropout** | 0 |
| **Quantization** | 4-bit |
| **Vision layers fine-tuned** | No (language layers only) |
| **Trainable params** | 29.8M / 4.3B (0.69%) |
| **Batch size** | 1 per device × 4 gradient accumulation = effective 4 |
| **Epochs** | 5 |
| **Learning rate** | 2e-4 (linear decay) |
| **Warmup steps** | 5 |
| **Optimizer** | adamw_8bit |
| **Max sequence length** | 1024 |
| **Failure oversampling** | 2× (48 samples after balancing) |
| **Data collator** | `UnslothVisionDataCollator(model, processor)` |

```python
from unsloth import FastVisionModel
```

## Analysis

### What Worked
- **State classification dramatically improved** (25% → 61%): the model learned the cooking vocabulary
- **Water-based boil-over detection works well**: pasta, ramen, soup, rice all correctly flagged
- **Training was fast and efficient**: 6:45 on 48 samples, 8.5GB VRAM peak
- **Loss converged cleanly**: 5.85 → 0.01, no instability

### What Didn't Work
- **5 false positives remain (danger→safe)**: these are the safety-critical failures
  - 3× FAILED/burnt states misclassified as safe (pasta, ramen, soup)
  - 2× boil-over missed (mapo_tofu, curry — thick non-water liquids)
- **"done" and "cold" confused**: model struggles to distinguish finished cooking from not-started
- **Curry consistently classified as "simmering"**: model defaults to this for brown liquids regardless of actual state
- **Inference slow**: 17-55s per image under LoRA (vs 1s baseline). Not optimized for real-time

### Root Causes
1. **Only 1 image per class**: 36 images total is extremely small. The model overfits to specific visual patterns rather than learning general concepts
2. **Non-water boil-overs are visually subtle**: thick curry/sauce spilling doesn't look like water overflowing
3. **"Burnt" is underrepresented**: only 6 burnt images, and synthetic images may not capture realistic char/burn patterns well
4. **Synthetic image quality**: Gemini-generated images are good but imperfect — some states look too similar

## Next Steps

1. **Generate more training variations** — especially for failed cases:
   - More curry/mapo_tofu boil-over images from different angles
   - More realistic burnt/charred food images
   - 3-5 variations per cell instead of 1
2. **Try vision layer fine-tuning** (`finetune_vision_layers=True`) — currently only language layers are trained
3. **Binary classifier approach** — instead of full state classification, train a simple safe/unsafe binary detector
4. **Real images** — collect actual kitchen photos to supplement synthetic data
5. **Merge LoRA for fast inference** — export merged model for deployment speed
